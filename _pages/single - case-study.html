A private insurance broker’s website gets a Design Upgrade

The client is frustrated and unhappy with his website because it looks unprofessional and disorganized. Moreover, the website neither inspires prospects nor reflects the company’s vision and services.

The Problem Image Gallery Technical Process Conclusion & Takeaway

The problem

The client is frustrated and unhappy with his website because it looks unprofessional and disorganized. Moreover, the website neither inspires prospects nor reflects the company’s vision and services.

Also over time, the website is not professionally managed, which further aggravates the unprofessional look. One important consequence is the loss of prospects who, after bouncing off the website reported that they “couldn’t find what they were looking for”. Others said, “This looks like an old-fashioned company that is run with desperation”.

For all these reasons, the client decided to get it redesigned by a “real professional”.

Technical Process

We created a solution in 5 major steps:

  1. A new foundation
  2. A polished and professional design
  3. An accessible and user-friendly design
  4. A website engineered with precision

Step 1: A new foundation

The client’s website lacks structure; there is no logical connection between pages, blog articles, and navigation elements: “There is no intuitiveness.” We start with the project’s cornerstone (technically called UX Research) which we complete in two steps:

  1. Study of the company’s business model and customer base
  2. Design of the information architecture
  3. Refinement of the website’s objectives
  4. Establishment of an attractive value proposition

Study of the company’s business model and customer base

It all starts with interviewing the client and learning as much as possible about the company, the business model, and the customers (this phase is technically called Empathy). We will later extract valuable insights from this information to create a value proposition, build a customer profile (technically called persona), and structure the new website information architecture.

The lack of access to real customers is challenging because it limits the research’s accuracy, which in turn limits the website’s. We worked around this limitation by building a persona based on insights collected from the client and online research.

Designing the information architecture

We completely reorganized the website information to ensure its easy and logical flow.

“We use insights collected from my interviews with the client to create keywords, categories, navigation items, widgets, and components that will all work in unison on the website to create a sense of flow and direction!“

We eliminate redundant information and regroup key insights into meaningful categories, simplifying the users’ experience (navigation and content displayed more intuitively and logically).

Refinement of the website’s objectives

Another problem that the client does not mention and that we decided to address is the lack of direction for the website. After careful analysis and client concertation, it was decided that the site’s objective would be to “educate users on the company’s services, and facilitate the discovery process to help them make informed life-insurance decisions.”

Establishment of an attractive value proposition

It was obvious from the start that the website was not catering enough to visitors’ self-interest, that’s why we decided to work on an attractive value proposition that would answer visitors’ intrinsic questions:

  • What is the offer?
  • Why should people care?
  • Why pick this offer and not a competitor’s

Once we clearly establish the value proposition, we embed it in every aspect of the design flow.

Step 2: A polished and professional design

The foundation that we just created (new information architecture, new website objectives, and value propositions) will now support the visual refinement that the client asked for in the first place. We achieved that visual refinement in two steps:

  • Wireframing of the experience
  • Selecting a visual style

Wireframing the experience

We create wireframes to shape the architecture and design objectives into components users can interact with (navigation, page elements, …).

From this point, we keep evaluating the wireframes to ensure their:

  • Compatibility with our team’s technical capabilities
  • Compliance with the new design objectives
  • Proper rendering on different screen sizes (technically called responsiveness)

Selecting a visual style

To create a sense of corporate sophistication and professionalism, we are using:

  • A mix of three design trends (Minimalism, Brutalism, Straight shapes, and flat design)
  • Neutral color tones (to create a sense of timeless elegance)
  • Straight shapes (to convey a sense of professionalism, precision, and order)

Image gallery

Step 3: An accessible and user-friendly design

By taking the time to make the design accessible, we are taking into consideration users that we tend to forget (people with low vision, with different screen sizes, or with different tastes). For this design, we are addressing four cases:

  • Contrast and color: Ensuring sufficient contrast between text and background colors
  • Responsiveness: Ensuring that the UI is adaptable to various screen sizes and orientations

Lighting Preferences: We have created a design’s dark mode version to help reduce users’ eye strain, particularly in low-light conditions.

Step 4: A website engineered with precision

We use WordPress to build the website. It combines all the necessary technical features (Content Management System, blogging, vast ecosystem of extensions), and more importantly because it is fully open-source.

Instead of relying on page builders (like Elementors), we create custom code because we aim for precision.

Pixel-perfect design integration

We develop the User Interface by creating markups and styles that are inspired by design files. We develop with a mobile-first approach, which consists in creating UI components and pages for small screens first, then gradually adapting the layout to fit medium and larger screens.

Rich functionality integration a-la-carte

We develop the website with a minimalist, lean, and our-of-the-box approach, we:

  • Code only what is necessary
  • Decide on a functionality/third-party system integration based on design needs
  • Kept everything to a minimum

Fast website rendering

Ensuring the website loads fast is a key aspect of good User Experience. To

  • Optimize images and database
  • Compress JavaScript and CSS files
  • Keep the code base light (theme and plug-ins)
  • Recommend quality hosting whenever we can

Conclusion

We started this journey with the goal of realigning a website with the vision of professionalism, order, and sophistication that the leadership has. During the research and analysis process, we discovered the website could bring more business if it could also “educate users on the company’s services, and facilitate their discovery process so that they could make informed life insurance decisions.”

We created the designs that reflect the leadership’s desires but also made sure they are accessible (as per modern standards). Once the time of development came, we created a pixel-perfect User Interface with rich functionalities that load fast.

Takeaway

We understood our client’s need to get a website that gives them prestige by reflecting elegance and sophistication, but we went further by ensuring users’ aspirations are met and that the website we created provides the best possible User Experience. We over-delivered on the client’s expectations and were very proud to do so.
